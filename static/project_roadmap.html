Project Roadmap

We plan to implement Bad Memory as our CSC309 term project, finishing most of the functionality by the Project 2 deadline on July 15. Project 3 will be devoted to addressing any issues with the performance and security of the website, including a stress-test of the functionality by both real and simulated users.

Product Backlog

The below product backlog ranks tasks from 1 to 3 in order of difficulty. We do not provide exact time estimates because time-to-implement may vary depending on the skill level of the teammate doing the task, and how much time is spent studying prior reference materials.

Backend Functionality

  • 3 Basic Database Design: tables for sentences, words, pages; operation log; indexing for search.
  • 2 REST API for Database: issue database operations through a RESTful API.
  • 2 Scrape Simple Webpages: obtain the text of a webpage and use it to populate the database with sentences and words.
  • 3 Store Webpages in Account: store a full copy of the webpage in the user's account.

Management Interface

  • 1 Account Signup & Management: standard features to create an account, manage password, log in & out.
  • 1 Sentence, Word, Page Tables: display tables of corresponding database entries.
  • 2 Search Functionality: allow to user to filter the tables according to various attributes.
  • 1 Manually Add Webpages & Sentences: form allowing the user to manually add a URL or text fragment for study.
  • 2 Add Webpages via Bookmarklet: provide a bookmarklet (with install instructions) that adds the current URL to Bad Memory.
  • 3 Heatmap UI: colour and highlight words according to their study status; click to add/remove from study list.
  • 3 Show Website Contents with Heatmap UI: display a web page, with the heatmap applied to the text (similarly to Google Translate).
  • 2 Obtain Definitions from Dictionary: hover in heatmap view for a definition; add dictionary entry to sample sentences.

Study Interface

  • 2 Generate Study Cards from Sentences: follow formats such as MCD or 10,000 sentences, depending on the user's preferences.
  • 2 SuperMemo2 Algorithm: provide REST operations to record a review for a given card.
  • 2 Provide Review Statistics and Study Goals: the study goals are used to adjust the suggested number of reviews/new cards per day.
  • 1 Display Review Statistics in Chart Form: these will be visible in the profile screen.
  • 2 Study Screen: display a card for review and allow the user to score themselves.

Sprint Backlog

Our first sprint will last a week, June 16 to June 23. We plan to implement the following functionality:

  • 3 Basic Database Design
  • 2 REST API for Database
  • 2 Scrape Simple Webpages
  • 1 Manually Add Webpages & Sentences
  • 2 Obtain Definitions from Dictionary

This provides a basic interface and the ability to use it to populate the database with real data for the further implementation sprints to make use of.
